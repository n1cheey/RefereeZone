# RefZone 3.0 Roadmap

## Product direction

RefZone 3.0 should evolve from a role-based referee portal into a federation-grade officiating operations platform.

Core principle:

- one active season workspace
- one source of truth per match
- one system for assignments, evaluation, reports, tests, finance, and communication

## Phase 1: Platform shell

- season-first navigation
- stronger operations dashboard
- role-aware module entry points
- local Supabase-first workflow
- mobile-safe API contracts

## Phase 2: Match Center

- one match page for referee crew, TO crew, statistic crew, reports, ranking, and media
- full event timeline and change history
- assignment confirmation states
- match protocol and video attachments

## Phase 3: Performance intelligence

- seasonal ranking dashboards
- referee and TO performance sheets
- report quality tracking
- season snapshots and trend analytics

## Phase 4: Education and compliance

- referee and TO tests
- certification flows
- retake logic
- seasonal eligibility checks

## Phase 5: Finance and governance

- match fee tracking
- payment status
- exports for federation finance
- member lifecycle and access governance

## Phase 6: Mobile foundation

- clean shared types
- role-safe API boundaries
- reusable domain logic
- offline-tolerant workflows for iOS and Android
